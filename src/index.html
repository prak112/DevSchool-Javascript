Breakpoints & Containers

Breakpoints are used to create responsive content, i.e, content layout changes based on the device-width (viewport size).

Content in the container responds as follows :

  • IF device-width or viewport size ≥ min-width, THEN content EXPANDS
  • IF device-width or viewport size < min-width, THEN content SHRINKS

The above content is in a default container class, .container, which adapts similar to container-sm

Grid System

Grid System Hierarchy

Container - Row - Column

  • Grid System (a.k.a, Twelve-Column Grid System) is based on containing 12 columns within the Grid (parent container).
  • Each column in the Grid occupies exactly 25% of the parent container.
  • In the below example, column class .col-sm-4 implies column occupies 4 of 12 columns
  • Column class .col-sm-4 also further implies responsive layout based on sm breakpoint

Grid System supports and responds to Breakpoints

  • Grid System classes can be defined with Breakpoints as .col-{breakpoint}-{col-width}.
  • Container and columns size themselves as per the min-width restrictions applied.
  • Columns on default class, .col, would elongate the content.
  • Columns on defined class, .col-sm-4 would occupy the exact space (4 of 12 columns & 25% of parent container)
  • On reaching their respective Breakpoint, switch the column from vertical to horizontal position.

Columns can be defined to have variable-width or equal-width. For example:

Only default container class, .container has access to customisation of:

  • Gutter - spaces in between columns
  • Nesting - nest columns inside a column
  • Column order
